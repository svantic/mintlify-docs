Svantic uses the W3C Trace Context specification for distributed tracing. When an agent dispatches a request to a remote capability, it injects `traceparent` and `baggage` headers so the receiving agent can join the same trace. The `@svantic/telemetry/propagation` subpath exports the utilities for both sides of this handoff: `ContextInjector` for outbound requests, `ContextExtractor` for inbound requests, and `BAGGAGE_KEYS` for the well-known Svantic baggage key names.

## The traceparent header

The `traceparent` header carries trace context in a fixed format:

```
00-{trace_id}-{span_id}-01
```

- `00` — W3C version, always `00`.
- `{trace_id}` — 32-character lowercase hex trace ID.
- `{span_id}` — 16-character lowercase hex span ID (the sender's current span).
- `01` — trace-flags byte; `01` means sampled.

Example:

```
traceparent: 00-0af7651916cd43dd8448eb211c80319c-b7ad6b7169203331-01
```

## The baggage header

The `baggage` header carries Svantic-specific context as comma-separated key-value pairs. Values are URL-encoded.

```
baggage: svantic.session_id=sess-123,svantic.tenant_id=acme
```

## Injecting headers (outbound)

Use `ContextInjector.inject()` to add `traceparent` and `baggage` to any outbound headers map before making an HTTP request.

```typescript
import { ContextInjector } from '@svantic/telemetry/propagation';

const headers: Record<string, string> = {};

ContextInjector.inject(headers, {
    trace_id: '0af7651916cd43dd8448eb211c80319c',
    span_id: 'b7ad6b7169203331',
    baggage: {
        'svantic.session_id': 'sess-123',
        'svantic.tenant_id': 'acme',
    },
});

// headers now contains:
//   traceparent: "00-0af7651916cd43dd8448eb211c80319c-b7ad6b7169203331-01"
//   baggage: "svantic.session_id=sess-123,svantic.tenant_id=acme"
```

`inject()` mutates the `headers` object in place. Trace and span IDs are normalised to lowercase hex in the header value regardless of the case you pass in.

<Warning>
  `inject()` throws if `trace_id` is not exactly 32 hex characters or `span_id` is not exactly 16 hex characters. Validate IDs before injecting or use IDs generated by `SvanticTracer`, which always produces correctly-sized IDs.
</Warning>

Use `tracer.get_current_context()` to get the current trace and span IDs from an active span:

```typescript
import { ContextInjector } from '@svantic/telemetry/propagation';

const ctx = tracer.get_current_context();
if (ctx) {
    ContextInjector.inject(outbound_headers, {
        trace_id: ctx.trace_id,
        span_id: ctx.span_id,
        baggage: {
            [BAGGAGE_KEYS.SESSION_ID]: current_session_id,
            [BAGGAGE_KEYS.TENANT_ID]: current_tenant_id,
        },
    });
}
```

## Extracting headers (inbound)

Use `ContextExtractor.extract()` to parse `traceparent` and `baggage` from headers you receive on an inbound request.

```typescript
import { ContextExtractor } from '@svantic/telemetry/propagation';

const ctx = ContextExtractor.extract(request.headers);

if (ctx) {
    // ctx.trace_id    — 32-char lowercase hex
    // ctx.span_id     — 16-char lowercase hex
    // ctx.trace_flags — integer decoded from the flags byte (1 = sampled)
    // ctx.baggage     — { 'svantic.session_id': 'sess-123', ... }
}
```

`extract()` returns `null` when:
- The `traceparent` header is absent.
- The header value doesn't have exactly four dash-separated fields.
- The version field is not `00`.
- The trace ID or span ID are not the correct hex lengths.

Header key matching is case-insensitive — `Traceparent`, `TRACEPARENT`, and `traceparent` all work.

## Joining a distributed trace

After extracting context, call `tracer.adopt_remote_context()` so local spans become children of the remote caller's span.

```typescript
import { ContextExtractor } from '@svantic/telemetry/propagation';

const ctx = ContextExtractor.extract(request.headers);

if (ctx) {
    tracer.adopt_remote_context(ctx.trace_id, ctx.span_id);
}

// All subsequent spans are now children of the remote span
const span = tracer.start_span('process-capability');
// span.parent_span_id === ctx.span_id
span.end();
```

The adopted context is cleared automatically when the last local span ends.

## Well-known baggage keys

The `BAGGAGE_KEYS` constant exports the canonical key names for Svantic context. Use these instead of raw strings to avoid typos.

```typescript
import { BAGGAGE_KEYS } from '@svantic/telemetry/propagation';
```

| Constant | Key string | Purpose |
|----------|-----------|---------|
| `BAGGAGE_KEYS.SESSION_ID` | `svantic.session_id` | Ties spans to a dashboard session |
| `BAGGAGE_KEYS.TENANT_ID` | `svantic.tenant_id` | Multi-tenancy scoping |
| `BAGGAGE_KEYS.INVOCATION_ID` | `svantic.invocation_id` | Links a specific tool call to its remote execution |
| `BAGGAGE_KEYS.AGENT_TYPE` | `svantic.agent_type` | Identifies the agent type in the DAG |
| `BAGGAGE_KEYS.INSTANCE_ID` | `svantic.instance_id` | Pinpoints the specific pod or process |

## Complete end-to-end example

The following shows the full propagation lifecycle across two agents.

<CodeGroup>

```typescript Calling agent (outbound)
import { SvanticTelemetry } from '@svantic/telemetry';
import { ContextInjector, BAGGAGE_KEYS } from '@svantic/telemetry/propagation';

const telemetry = SvanticTelemetry.init({
    service_name: 'orchestrator',
    export: {
        endpoint: 'http://localhost:7001/telemetry/otlp',
        auth: { type: 'api_key', key: 'tenant-secret' },
    },
});

const tracer = telemetry.get_tracer();
const span = tracer.start_span('dispatch-to-summarizer');

const headers: Record<string, string> = {
    'Content-Type': 'application/json',
};

const ctx = tracer.get_current_context();
if (ctx) {
    ContextInjector.inject(headers, {
        trace_id: ctx.trace_id,
        span_id: ctx.span_id,
        baggage: {
            [BAGGAGE_KEYS.SESSION_ID]: 'sess-abc',
            [BAGGAGE_KEYS.TENANT_ID]: 'acme',
            [BAGGAGE_KEYS.INVOCATION_ID]: 'inv-001',
        },
    });
}

await fetch('http://summarizer:8080/run', {
    method: 'POST',
    headers,
    body: JSON.stringify({ text: document_content }),
});

span.end();
await telemetry.shutdown();
```

```typescript Receiving agent (inbound)
import { SvanticTelemetry } from '@svantic/telemetry';
import { ContextExtractor, BAGGAGE_KEYS } from '@svantic/telemetry/propagation';

const telemetry = SvanticTelemetry.init({
    service_name: 'summarizer',
    export: {
        endpoint: 'http://localhost:7001/telemetry/otlp',
        auth: { type: 'api_key', key: 'tenant-secret' },
    },
});

const tracer = telemetry.get_tracer();

// Inside your HTTP request handler:
const ctx = ContextExtractor.extract(request.headers);

if (ctx) {
    tracer.adopt_remote_context(ctx.trace_id, ctx.span_id);

    const session_id = ctx.baggage[BAGGAGE_KEYS.SESSION_ID];
    const tenant_id  = ctx.baggage[BAGGAGE_KEYS.TENANT_ID];
    // use session_id and tenant_id for scoping
}

const span = tracer.start_span('summarize');
// span is now a child of the orchestrator's span
span.end();
```

</CodeGroup>

## How the Svantic mesh uses propagation

When the mesh dispatches a request to a remote agent via `RemoteCapabilityProxy`, it:

1. Reads the active trace and span IDs from the current context.
2. Calls `ContextInjector.inject()` to add `traceparent` to the outbound headers.
3. Adds the session ID, tenant ID, and invocation ID to the `baggage` header using the `BAGGAGE_KEYS` constants.

The remote agent receives the headers and calls `adopt_remote_context()` (or uses `A2aInstrumentation.extract_context()`) to attach local spans to the mesh-initiated trace. This creates a single connected trace across multiple agent hops without any manual trace ID passing through your application code.

<Tip>
  Use `A2aInstrumentation.extract_context()` in your agent's request handler as a one-liner — it handles the `ContextExtractor.extract()` and `tracer.adopt_remote_context()` calls together. See [Tracing](/telemetry/tracing) for the full example.
</Tip>
